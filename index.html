So, what is TypeScript? TypeScript is an opensource programming language developed and maintained by Microsoft. TypeScript is typed superset of Javascript, which compiles to Javascript. You can find documentation on the official site www.typescriptlang.org üìù

Static type checking


						let two: number = 2;
						two = '2'; // Error: cannot assign 'string' to 'number'
  • Allows IDEs to perform live error checks
  • Exposes auto-completion and code hinting
  • Easier refactor process
  • Less unit testing
The main feature of Typescript is static type checking. It allows IDEs to perform live error checks, expose auto-completion and code hinting. It simplifies refactor and unint testing processes. üìù

Interface


						interface IName {
							propName1[:type]; // required property
							propName2?[:type]; // optional property
							[propName: string][:type]; // extra properties
						}
  • Design time feature (no extra code would be emitted to JS file
  • Errors being shown when interface signature and implementation doesn‚Äôt match
One of TypeScript‚Äôs core principles is that type checking focuses on the shape that values have. In TypeScript, interfaces fill the role of naming these types, and are a powerful way of defining contracts within your code as well as contracts with code outside of your project. Errors are immediately shown when interface signature and implementation doesn‚Äôt match so developer can fix the problem before running the code. It is worth noting that interfaces are design time feature and no extra code would be emitted to JS file. üìù
